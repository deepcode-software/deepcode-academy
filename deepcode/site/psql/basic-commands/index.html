‚úÖ DATA TYPES

üìå PostgreSQLda ma'lumot turlari (data types) ustunlarda saqlanadigan ma'lumotlarning turini belgilaydi va ular bilan qanday ishlash mumkinligini aniqlaydi.

    -- Talabaning ismini saqlaydi

‚ú≥Ô∏è BIGINT

üìå BIGINT (int8) ‚Äî bu 8 baytli butun son turi bo‚Äòlib, juda katta butun sonlarni saqlash uchun ishlatiladi.

üéØ Oddiy BIGINT turidagi ustun yaratish uchun Example jadvali.

-- Bu jadvalda faqat bitta ustun bor: big_num

    big_num BIGINT

üéØ Bank tranzaktsiyalarini saqlash uchun jadval yaratish

    -- Har bir tranzaktsiyaga noyob raqam beriladi

    -- description ustuni yaratilmoqda

    description VARCHAR(100)

‚ú≥Ô∏è SMALLINT

üìå SMALLINT (int2) ‚Äî bu 2 baytli butun son turi bo‚Äòlib, kichik diapazondagi butun sonlarni saqlash uchun ishlatiladi.

üéØ Oddiy SMALLINT turidagi ustun yaratish uchun example jadvali:

-- Bu jadvalda faqat bitta ustun bor: small_num
-- small_num ustuni SMALLINT (kichik butun son) tipida bo‚Äòladi

    small_num SMALLINT
);

üéØ Xodimlarning lavozim darajasini saqlash uchun.

‚ú≥Ô∏è DECIMAL OR NUMERIC

üìå DECIMAL yoki NUMERIC ‚Äî bu aniq o‚Äònlik kasr sonlarni saqlash uchun ishlatiladigan ma'lumot turi. Bu turda aniqlik (precision) va kasr sonlar soni (scale) aniq belgilanadi.

‚ú≥Ô∏è REAL

üìå REAL ‚Äî bu 4 baytli haqiqiy son (floating point) turi bo‚Äòlib, o‚Äònlik kasr sonlarni taxminiy aniqlikda saqlaydi.

‚ú≥Ô∏è DOUBLE PRECISION

DOUBLE PRECISION ‚Äî bu 8 baytli aniqroq suzuvchi nuqtali haqiqiy sonlarni saqlash uchun ishlatiladigan ma'lumot turi, katta va kichik onlik sonlarni aniqlik bilan saqlaydi.

    -- Bu ustun DOUBLE PRECISION tipida bo‚Äòladi

‚ùáÔ∏è TEXT TYPE

‚ú≥Ô∏è VARCHAR(N)

VARCHAR(n) ‚Äî bu o‚Äòzgaruvchan uzunlikdagi matn turi, matn uzunligi qancha bo‚Äòlsa, shuncha joy egallaydi, lekin maksimal uzunlik n dan oshmaydi.

‚ú≥Ô∏è TEXT

    -- TEXT - bu ustun cheksiz uzunlikdagi matn saqlaydi
    -- Matn uzunligi oldindan belgilab qo‚Äòyilmaydi
    -- Juda katta (katta hajmli) matnlar uchun qulay

‚ùáÔ∏è DATE AND TIME TYPES

‚ú≥Ô∏è DATE

DATE - bu faqat sana (yil, oy, kun) ni saqlaydigan ma'lumotlar turi. DATE turi vaqtni (soat, daqiqa, soniya) o‚Äòz ichiga olmaydi ‚Äî faqatgina sana qismi saqlanadi.

‚ú≥Ô∏è TIME

TIME ‚Äî bu faqat vaqt (soat, daqiqa, soniya) ni saqlaydigan ma'lumotlar turi. Bu turda sana (yil, oy, kun) saqlanmaydi ‚Äî faqat kun ichidagi vaqt ifodalanadi.

‚ú≥Ô∏è TIMESTAMP

    -- TIMESTAMP - bu ustun sana va vaqtni birgalikda saqlaydi

TIMESTAMPTZ (Timestamp with Time Zone) ‚Äî bu sana va vaqtni soat mintaqasi bilan birga saqlaydigan ma'lumotlar turi. ‚ú≥Ô∏è INTERVAL

INTERVAL ‚Äî bu vaqt oralig‚Äòini (davrni) saqlovchi ma'lumotlar turi. Masalan: 1 kun, 3 soat, 2 oy 15 kun, 1 yil 5 oy 10 kun 4 soat 30 daqiqa kabi qiymatlar saqlanishi mumkin. Bu tur vaqtlar orasidagi farqni ifodalash yoki vaqt ustiga qo‚Äòshish/ayirishda ishlatiladi.

‚ùáÔ∏è BOOLEAN TYPE

BOOLEAN ‚Äî bu mantiqiy (true/false) qiymatlarni saqlovchi ma'lumotlar turi. Faqatgina TRUE, FALSE yoki NULL qiymatlarni qabul qiladi. Ko‚Äòpincha holatni bildiruvchi ustunlarda ishlatiladi (masalan: foydalanuvchi faolmi yoki yo‚Äòqmi).

PRIMARY KEYS AND CONSTRAINTS

[!NOTE] Ma'lumotlar bazasida asosiy kalit (primary key) va cheklovlar (constraints) ma'lumotlarning to'g'ri va tartibli saqlanishini ta'minlash uchun ishlatiladi.

Primary key

Primary key jadvaldagi har bir qatorni yagona tarzda identifikatsiya qiladi. U shuni ta'minlaydiki, jadvaldagi ikkita qator bir xil asosiy kalit qiymatiga ega bo'lmaydi va asosiy kalit hech qachon NULL bo'lmaydi.

  • Characteristics of a Primary Key:
  • Uniqueness: Har bir qator uchun asosiy kalit qiymati takrorlanmas bo'lishi kerak.
  • Non-null: Primary key ustunida NULL qiymatlari bo'lishi mumkin emas.
  • Single-column or Composite: Asosiy kalit bitta ustundan yoki bir nechta ustunlarning birikmasidan iborat bo'lishi mumkin.

Example:

CREATE TABLE Students (
    StudentID INT PRIMARY KEY,
    Name VARCHAR(100),
    Age INT
);

Bu yerda StudentID asosiy kalit bo'lib, har bir talabaning noyob IDga ega bo'lishini ta'minlaydi.

Composite Primary Key Example:

CREATE TABLE Enrollments (
    StudentID INT,
    CourseID INT,
    PRIMARY KEY (StudentID, CourseID)
);

Bu jadvalda StudentID va CourseID birgalikda birikma asosiy kalit sifatida ishlatiladi, ya'ni har bir talaba bitta kursga faqat bir marta yozilishi mumkin.

Constraints

[!NOTE] Constraints jadval ustunlariga qo'llaniladigan qoidalar bo'lib, ular ma'lumotlarning to'g'ri va tartibli bo'lishini ta'minlaydi.

Types of Constraints: 1. NOT NULL

Ustunda NULL qiymat bo'lishiga ruxsat bermaydi.

Example:

CREATE TABLE Employees (
    EmployeeID INT PRIMARY KEY,
    Name VARCHAR(100) NOT NULL
);
  1. UNIQUE

Ustundagi barcha qiymatlar takrorlanmas bo'lishini ta'minlaydi.

Example:

CREATE TABLE Products (
    ProductID INT PRIMARY KEY,
    SKU VARCHAR(50) UNIQUE
);
  1. PRIMARY KEY

U NOT NULL va UNIQUE qoidalarini birlashtirib, har bir qatorni noyob identifikatsiyalaydi.

  1. FOREIGN KEY

Jadvalni boshqa jadval bilan bog'laydi, boshqa jadvalning asosiy kalitiga ishora qiladi.

Example:

CREATE TABLE Orders (
    OrderID INT PRIMARY KEY,
    CustomerID INT,
    FOREIGN KEY (CustomerID) REFERENCES Customers(CustomerID)
);
  1. CHECK

Ustundagi qiymatlarning ma'lum bir shartga mos kelishini tekshiradi.

Example:

CREATE TABLE Accounts (
    AccountID INT PRIMARY KEY,
    Balance DECIMAL(10, 2) CHECK (Balance >= 0)
);
  1. DEFAULT

Agar kiritishda qiymat berilmasa, ustun uchun standart qiymatni belgilaydi.

Example:

CREATE TABLE Users (
    UserID INT PRIMARY KEY,
    RegistrationDate DATE DEFAULT CURRENT_DATE
);
