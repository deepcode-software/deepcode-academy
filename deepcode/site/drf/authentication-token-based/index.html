üß© 7-DARS: AUTENTIFIKATSIYA (TOKEN-GA ASOSLANGAN AUTENTIFIKATSIYA)

Bu darsda Django REST Framework (DRF) yordamida token-ga asoslangan autentifikatsiyani sozlashni bosqichma-bosqich o'rganamiz. Token autentifikatsiyasi API ga xavfsiz kirishni ta'minlaydi. Har bir qadam tushunarli va faqat token autentifikatsiyasiga qaratilgan bo'ladi. Oldingi darslarda sozlangan myproject loyihasi va myapp ilovasidagi Task modeli asosida davom etamiz.

‚úÖ 1. TAYYORLOV ISHLARI

üìå Loyiha va ilova allaqachon sozlangan deb hisoblaymiz (myproject, myapp, va Task modeli). Quyidagi sozlamalar mavjud bo'lishi kerak: - Task modeli myapp/models.py faylida belgilangan. - TaskSerializer myapp/serializers.py faylida yaratilgan. - TaskViewSet myapp/views.py faylida sozlangan. - DefaultRouter bilan /tasks/ endpointi myproject/urls.py faylida ro'yxatdan o'tkazilgan. üìå Agar bu sozlamalar hali amalga oshirilmagan bo'lsa, avvalgi darslarga qayting.

‚úÖ 2. TOKEN AUTENTIFIKATSIYASINI TUSHUNISH

üìå DRFning token-ga asoslangan autentifikatsiyasi har bir foydalanuvchiga noyob token beradi. Ushbu token API so'rovlarida foydalanuvchi identifikatsiyasi uchun ishlatiladi. Tokenlar HTTP sarlavhalarida (Authorization) yuboriladi. üìå Afzalliklari: - Oddiy va xavfsiz. - Har bir foydalanuvchi uchun alohida token. - Tokenni osongina o'chirish yoki yangilash mumkin.

‚úÖ 3. TOKEN AUTENTIFIKATSIYASINI SOZLASH

üìå DRFning o'rnatilgan TokenAuthentication sinfidan foydalanamiz.

‚ùáÔ∏è DRF autentifikatsiya paketini o'rnatish:

üìå Token autentifikatsiyasi uchun qo'shimcha paket o'rnatish shart emas, chunki u DRF bilan birga keladi. Ammo loyihada rest_framework o'rnatilganligini tekshiring:

pip install djangorestframework

‚ùáÔ∏è Token autentifikatsiyasini loyihaga qo'shish:

üìå myproject/settings.py faylini yangilang va REST_FRAMEWORK sozlamalarini qo'shing:

INSTALLED_APPS = [
    ...
    'rest_framework',
    'rest_framework.authtoken',  # Token autentifikatsiyasi uchun
    'myapp',
]

REST_FRAMEWORK = {
    'DEFAULT_AUTHENTICATION_CLASSES': [
        'rest_framework.authentication.TokenAuthentication',
    ],
    'DEFAULT_PERMISSION_CLASSES': [
        'rest_framework.permissions.IsAuthenticated',  # Faqat autentifikatsiya qilingan foydalanuvchilar uchun
    ],
}

‚ùáÔ∏è Migratsiyalarni qo'llash:

üìå Token autentifikatsiyasi uchun ma'lumotlar bazasida authtoken_token jadvali yaratilishi kerak:

python manage.py migrate

‚úÖ 4. VIEWSET NI AUTENTIFIKATSIYA BILAN YANGILASH

üìå myapp/views.py faylida TaskViewSet ni autentifikatsiya va ruxsatlar bilan yangilaymiz:

from rest_framework import viewsets
from rest_framework.permissions import IsAuthenticated
from .models import Task
from .serializers import TaskSerializer

class TaskViewSet(viewsets.ModelViewSet):
    queryset = Task.objects.all()
    serializer_class = TaskSerializer
    permission_classes = [IsAuthenticated]  # Faqat autentifikatsiya qilingan foydalanuvchilar

üìå Tushuntirish: - permission_classes = [IsAuthenticated]: Faqat token bilan autentifikatsiya qilingan foydalanuvchilar API ga kira oladi. - TaskViewSet oldingi darsda sozlangan bo'lib, CRUD operatsiyalarini qo'llab-quvvatlaydi.

‚úÖ 5. ROUTER BILAN URL SOZLASH

üìå myproject/urls.py faylida URL marshrutlari oldingi darsdagidek qoladi, lekin token olish uchun qo'shimcha endpoint qo'shamiz:

from django.urls import path, include
from rest_framework.routers import DefaultRouter
from rest_framework.authtoken.views import obtain_auth_token
from myapp.views import TaskViewSet

router = DefaultRouter()
router.register(r'tasks', TaskViewSet)

urlpatterns = [
    path('', include(router.urls)),
    path('api-token-auth/', obtain_auth_token, name='api_token_auth'),  # Token olish uchun
]

üìå Tushuntirish: - obtain_auth_token: Foydalanuvchi login va parol yordamida token olishi uchun endpoint. - /tasks/ va /tasks/<pk>/ endpointlari TaskViewSet uchun ishlaydi.

‚úÖ 6. FOYDALANUVCHI VA TOKEN YARATISH

üìå Token autentifikatsiyasi uchun foydalanuvchi va token yaratamiz.

‚ùáÔ∏è Foydalanuvchi yaratish:

üìå Admin paneli orqali yoki buyruq satri yordamida foydalanuvchi yarating:

python manage.py createsuperuser

üìå Masalan: username: testuser, password: testpass123.

‚ùáÔ∏è Token yaratish:

üìå Token olish uchun /api-token-auth/ endpointiga POST so'rov yuboring:

curl -X POST http://127.0.0.1:8000/api-token-auth/ -d "username=testuser&password=testpass123"

üìå Javob sifatida token qaytadi, masalan:

{"token": "9944b09199c62bcf9418ad846dd0e4bbafc6ee0b"}

üìå Ushbu tokenni saqlang, chunki u API so'rovlarida ishlatiladi.

‚úÖ 7. AUTENTIFIKATSIYA BILAN API NI SINAB KO'RISH

üìå Serverni ishga tushiring:

python manage.py runserver

üìå API so'rovlarida tokenni Authorization sarlavhasida yuboring: Token <token>.

‚ùáÔ∏è Create (Yangi vazifa qo'shish):

üìå Manzil: http://127.0.0.1:8000/tasks/
üìå Usul: POST
üìå Masalan (Postman orqali):

curl -X POST http://127.0.0.1:8000/tasks/ -H "Authorization: Token 9944b09199c62bcf9418ad846dd0e4bbafc6ee0b" -H "Content-Type: application/json" -d '{"title": "Yangi vazifa", "description": "Bu test vazifa", "completed": false}'

üìå Token bo'lmasa, 401 Unauthorized xatosi qaytadi.

‚ùáÔ∏è Read (Ma'lumotlarni o'qish):

  • Barcha vazifalarni ko'rish:
    üìå Manzil: http://127.0.0.1:8000/tasks/
    üìå Usul: GET
    üìå Masalan: bash curl http://127.0.0.1:8000/tasks/ -H "Authorization: Token 9944b09199c62bcf9418ad846dd0e4bbafc6ee0b"

  • Bitta vazifani ko'rish:
    üìå Manzil: http://127.0.0.1:8000/tasks/1/
    üìå Usul: GET
    üìå Masalan: bash curl http://127.0.0.1:8000/tasks/1/ -H "Authorization: Token 9944b09199c62bcf9418ad846dd0e4bbafc6ee0b"

‚ùáÔ∏è Update (Ma'lumotlarni yangilash):

üìå Manzil: http://127.0.0.1:8000/tasks/1/
üìå Usul: PATCH
üìå Masalan:

curl -X PATCH http://127.0.0.1:8000/tasks/1/ -H "Authorization: Token 9944b09199c62bcf9418ad846dd0e4bbafc6ee0b" -H "Content-Type: application/json" -d '{"completed": true}'

‚ùáÔ∏è Delete (Ma'lumotlarni o'chirish):

üìå Manzil: http://127.0.0.1:8000/tasks/1/
üìå Usul: DELETE
üìå Masalan:

curl -X DELETE http://127.0.0.1:8000/tasks/1/ -H "Authorization: Token 9944b09199c62bcf9418ad846dd0e4bbafc6ee0b"

‚úÖ 8. ADMIN PANELI ORQALI SINOV (IXTIYORIY)

üìå Ma'lumotlarni qo'lda sinash uchun admin panelidan foydalaning:

‚ùáÔ∏è Admin foydalanuvchisini tekshirish:

üìå Agar hali yaratilmagan bo'lsa:

python manage.py createsuperuser

‚ùáÔ∏è Modelni admin panelida ro'yxatdan o'tkazish:

üìå myapp/admin.py faylida quyidagi kod bo'lishi kerak:

from django.contrib import admin
from .models import Task

admin.site.register(Task)

üìå http://127.0.0.1:8000/admin/ manzilida admin paneliga kiring va vazifalarni qo'shing, tahrirlang yoki o'chiring.

‚úÖ 9. SINOV UCHUN MASALALAR

üìå Token autentifikatsiyasini sinash uchun quyidagi amallarni bajarib ko'ring: 1. /api-token-auth/ manzilidan token oling. 2. Token bilan /tasks/ manzilida yangi vazifa qo'shing. 3. Tokensiz so'rov yuboring va 401 Unauthorized xatosini oling. 4. /tasks/1/ manzilida vazifani yangilang yoki o'chiring.
