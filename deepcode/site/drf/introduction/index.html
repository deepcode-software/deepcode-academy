‚úÖ DJANGO REST FRAMEWORK (DRF) NIMA?

üìå DRF ‚Äì bu Django REST Framework degan so‚Äòzlarning qisqartmasi. Bu Django frameworki ustiga qo‚Äòshimcha kutubxona bo‚Äòlib, u orqali biz API (Application Programming Interface) yaratamiz.

üìå DRF yordamida biz backend (ma‚Äôlumotlar bazasi bilan ishlovchi qism) bilan frontend yoki mobil ilovalar o‚Äòrtasida JSON formatidagi muloqotni tashkil qilamiz.

üìå Bu kutubxona nafaqat CRUD amallarni amalga oshirish, balki authentication, permission, pagination, throttling, filtering, versioning kabi ilg‚Äòor funksiyalarni ham osonlashtiradi.

‚úÖ DRF NIMA UCHUN KERAK?

üìå Django odatda veb sahifalar (HTML) bilan ishlashga moslashgan. Ammo ko‚Äòplab zamonaviy ilovalar (masalan, mobil ilovalar, frontend (React, Vue)) uchun API (Application Programming Interface) talab qilinadi.

üìå DRF aynan shunday holatlarda:

  • üîÅ Django ma'lumotlarini JSON formatida frontend/mobil ilovaga uzatishda
  • ‚úÖ CRUD (Create, Read, Update, Delete) amallarini API orqali bajarishda
  • üîê Token/Session asosida authentication va permission larni tashkil qilishda
  • üß™ API larni tez, xavfsiz va testga yaroqli qilishda juda foydalidir.

‚úÖ REST API NIMA?

üìå REST (Representational State Transfer) ‚Äî bu internet orqali resurslar (odatda ma‚Äôlumotlar) bilan ishlash usuli.

üìå REST API bu:

  • GET ‚Äî ma'lumot olish
  • POST ‚Äî ma'lumot yaratish
  • PUT/PATCH ‚Äî ma'lumotni yangilash
  • DELETE ‚Äî ma'lumotni o‚Äòchirish

‚ú≥Ô∏è 2. Serializers

üìå DRF da serializer ‚Äì bu Django modelidagi ma‚Äôlumotlarni JSON, XML yoki boshqa formatlarga o‚Äòtkazish uchun ishlatiladigan vosita. Shu bilan birga, u ma‚Äôlumotlarni tekshirish (validation) va yaratish/yangilash (create/update) imkonini beradi. 

# serializers.py

from rest_framework import serializers
from .models import Student

class StudentSerializer(serializers.ModelSerializer):
    class Meta:
        model = Student
        fields = ['id', 'name', 'age', 'email']

‚ú≥Ô∏è 3. Views

# views.py

from rest_framework.decorators import api_view
from rest_framework.response import Response
from .models import Student
from .serializers import StudentSerializer

@api_view(['GET'])
def student_list(request):
    students = Student.objects.all()          # Barcha studentlarni olish
    serializer = StudentSerializer(students, many=True)  # JSONga aylantirish
    return Response(serializer.data)          # Foydalanuvchiga yuborish
  • many=True ‚Üí ko‚Äòp obyektlarni JSON formatga aylantirish uchun
  • serializer.data ‚Üí JSON ma‚Äôlumotlar

‚ú≥Ô∏è 4. urls

‚ú≥Ô∏è POST so‚Äòrov bilan yangi student qo‚Äòshish

@api_view(['GET', 'POST'])
def student_list(request):
    if request.method == 'GET':
        students = Student.objects.all()
        serializer = StudentSerializer(students, many=True)
        return Response(serializer.data)

    elif request.method == 'POST':
        serializer = StudentSerializer(data=request.data)  # Foydalanuvchidan ma'lumot olish
        if serializer.is_valid():                           # Tekshirish
            serializer.save()                               # Bazaga saqlash
            return Response(serializer.data, status=201)
        return Response(serializer.errors, status=400)     # Xato bo‚Äòlsa
