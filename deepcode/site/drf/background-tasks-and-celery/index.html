üß© 14-DARS: FON VAZIFALARI VA CELERY BILAN ISHLASH

Bu darsda Django REST Framework (DRF) yordamida fon vazifalarini (background tasks) va Celery bilan ishlashni bosqichma-bosqich o'rganamiz. Celery uzoq davom etadigan yoki resurs talab qiladigan vazifalarni asinxron tarzda bajarish uchun ishlatiladi. Har bir qadam tushunarli va faqat fon vazifalari va Celery ga qaratilgan bo'ladi. Oldingi darslarda sozlangan myproject loyihasi va myapp ilovasidagi Task modeli, JWT autentifikatsiyasi, ruxsatlar va signallar asosida davom etamiz.

‚úÖ 1. TAYYORLOV ISHLARI

üìå Loyiha va ilova allaqachon sozlangan deb hisoblaymiz (myproject, myapp, va Task modeli). Quyidagi sozlamalar mavjud bo'lishi kerak: - Task modeli myapp/models.py faylida belgilangan (shu jumladan owner maydoni). - TaskSerializer myapp/serializers.py faylida yaratilgan. - TaskViewSet myapp/views.py faylida sozlangan (JWT autentifikatsiyasi, ruxsatlar, filtrlash va sahifalash bilan). - Signallar myapp/signals.py faylida sozlangan. - /tasks/ endpointi myproject/urls.py faylida sozlangan. üìå Agar bu sozlamalar hali amalga oshirilmagan bo'lsa, avvalgi darslarga qayting.

‚úÖ 2. CELERY VA FON VAZIFALARINI TUSHUNISH

üìå Celery ‚Äî Python loyihalarida asinxron vazifalarni boshqarish uchun kuchli vosita. U uzoq vaqt talab qiladigan vazifalarni (masalan, log yozish, email yuborish) asosiy ilova jarayonidan ajratib, fon rejimida bajaradi. üìå Fon vazifalari: - API so'rovlarining tezligini oshiradi. - Serverni ortiqcha yukdan himoya qiladi. - Vazifalarni parallel ravishda bajarish imkonini beradi. üìå Celery ishlatish uchun Redis yoki RabbitMQ kabi xabar brokeri talab qilinadi. Ushbu darsda Redis dan foydalanamiz.

‚úÖ 3. CELERY VA REDIS NI O'RNATISH

üìå Quyidagi paketlarni o'rnatamiz:

pip install celery redis

üìå Redis ni o'rnatish (masalan, Ubuntu uchun):

sudo apt-get install redis-server

üìå Redis serverini ishga tushiring va uning ishlayotganligini tekshiring:

redis-server
redis-cli ping  # Javob: PONG

‚úÖ 4. CELERY SOZLAMALARINI QO'SHISH

üìå myproject papkasida celery.py faylini yarating va quyidagi kodni qo'shing:

import os
from celery import Celery

os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'myproject.settings')

app = Celery('myproject')

app.config_from_object('django.conf:settings', namespace='CELERY')

app.autodiscover_tasks()

üìå Tushuntirish: - Celery('myproject'): Celery ilovasini yaratadi. - config_from_object: Django sozlamalaridan Celery konfiguratsiyasini oladi. - autodiscover_tasks: Ilovadagi vazifalarni avtomatik topadi.

üìå myproject/__init__.py faylini yangilang:

from .celery import app as celery_app

__all__ = ('celery_app',)

‚úÖ 6. FON VAZIFASINI YARATISH

üìå myapp/tasks.py faylini yarating va vazifa yaratilganda log yozish uchun fon vazifasini qo'shing:

from celery import shared_task
import logging

logger = logging.getLogger(__name__)

@shared_task
def log_task_creation(task_id, task_title, owner_username):
    logger.info(f"[Celery] Yangi vazifa yaratildi: {task_title} (ID: {task_id}, Owner: {owner_username})")

üìå Tushuntirish: - @shared_task: Celery vazifasini belgilaydi. - log_task_creation: Vazifa yaratilganda log yozadi.

‚úÖ 7. SIGNALLARNI CELERY BILAN INTEGRATSIYA QILISH

üìå myapp/signals.py faylini yangilaymiz va vazifa yaratilganda Celery vazifasini chaqiramiz:

from django.db.models.signals import post_save, post_delete
from django.dispatch import receiver
from .models import Task
from .tasks import log_task_creation
import logging

logger = logging.getLogger(__name__)

@receiver(post_save, sender=Task)
def handle_task_creation(sender, instance, created, **kwargs):
    if created:
        # Celery vazifasini fon rejimida ishga tushirish
        log_task_creation.delay(instance.id, instance.title, instance.owner.username)
    else:
        logger.info(f"Vazifa yangilandi: {instance.title} (ID: {instance.id}, Owner: {instance.owner.username})")

@receiver(post_delete, sender=Task)
def log_task_deletion(sender, instance, **kwargs):
    logger.info(f"Vazifa o'chirildi: {instance.title} (ID: {instance.id}, Owner: {instance.owner.username})")

üìå Tushuntirish: - log_task_creation.delay: Celery vazifasini asinxron tarzda ishga tushiradi. - post_delete signali o'zgarmagan holda qoldi.

‚úÖ 8. CELERY ISHGA TUSHIRISH

üìå Celery worker ni ishga tushirish uchun yangi terminal oching va loyiha papkasida quyidagi buyruqni bajaring:

celery -A myproject worker --loglevel=info

üìå Eslatma: Redis serveri ishlayotgan bo'lishi kerak.

‚úÖ 9. VIEWSET NI TEKSHIRISH

üìå myapp/views.py faylida TaskViewSet oldingi darsdagidek qoladi:

from rest_framework import viewsets
from django_filters.rest_framework import DjangoFilterBackend
from rest_framework.filters import SearchFilter, OrderingFilter
from .models import Task
from .serializers import TaskSerializer
from .permissions import IsOwnerOrReadOnly
from .pagination import CustomTaskPagination
from rest_framework.permissions import IsAuthenticated

class TaskViewSet(viewsets.ModelViewSet):
    queryset = Task.objects.all()
    serializer_class = TaskSerializer
    permission_classes = [IsAuthenticated, IsOwnerOrReadOnly]
    filter_backends = [DjangoFilterBackend, SearchFilter, OrderingFilter]
    filterset_fields = ['completed', 'owner']
    search_fields = ['title', 'description']
    ordering_fields = ['created_at', 'completed']
    ordering = ['created_at']
    pagination_class = CustomTaskPagination

    def perform_create(self, serializer):
        serializer.save(owner=self.request.user)

‚úÖ 10. URL SOZLASH

üìå myproject/urls.py faylida URL marshrutlari oldingi darsdagidek qoladi:

from django.urls import path, include
from rest_framework.routers import DefaultRouter
from rest_framework_simplejwt.views import TokenObtainPairView, TokenRefreshView
from myapp.views import TaskViewSet

router = DefaultRouter()
router.register(r'tasks', TaskViewSet)

urlpatterns = [
    path('', include(router.urls)),
    path('api/token/', TokenObtainPairView.as_view(), name='token_obtain_pair'),
    path('api/token/refresh/', TokenRefreshView.as_view(), name='token_refresh'),
]

‚úÖ 11. CELERY VAZIFALARINI SINAB KO'RISH

üìå Serverni ishga tushiring:

python manage.py runserver

üìå Celery worker ni alohida terminalda ishga tushiring:

celery -A myproject worker --loglevel=info

‚ùáÔ∏è Foydalanuvchi va token yaratish:

üìå Agar hali yaratilmagan bo'lsa, foydalanuvchi yarating:

python manage.py createsuperuser  # username: user1, password: pass123

üìå JWT token oling:

curl -X POST http://127.0.0.1:8000/api/token/ -H "Content-Type: application/json" -d '{"username": "user1", "password": "pass123"}'

‚ùáÔ∏è Vazifa yaratish va Celery sinovi:

üìå Manzil: http://127.0.0.1:8000/tasks/
üìå Usul: POST 

curl -X POST http://127.0.0.1:8000/tasks/ -H "Authorization: Bearer <access_token>" -H "Content-Type: application/json" -d '{"title": "Yangi vazifa", "description": "Test vazifa"}'

üìå Celery worker terminalida yoki debug.log faylida quyidagi logni ko'rasiz:

[Celery] Yangi vazifa yaratildi: Yangi vazifa (ID: 1, Owner: user1)

‚ùáÔ∏è Vazifa yangilash va signal sinovi:

üìå Manzil: http://127.0.0.1:8000/tasks/1/
üìå Usul: PATCH 

curl -X PATCH http://127.0.0.1:8000/tasks/1/ -H "Authorization: Bearer <access_token>" -H "Content-Type: application/json" -d '{"completed": true}'

üìå Log (Celery emas, signal orqali):

Vazifa yangilandi: Yangi vazifa (ID: 1, Owner: user1)

‚ùáÔ∏è Vazifa o'chirish va signal sinovi:

üìå Manzil: http://127.0.0.1:8000/tasks/1/
üìå Usul: DELETE 

curl -X DELETE http://127.0.0.1:8000/tasks/1/ -H "Authorization: Bearer <access_token>"

üìå Log:

Vazifa o'chirildi: Yangi vazifa (ID: 1, Owner: user1)

‚úÖ 12. ADMIN PANELI ORQALI SINOV (IXTIYORIY)

üìå Ma'lumotlarni qo'lda sinash uchun admin panelidan foydalaning:

‚ùáÔ∏è Admin foydalanuvchisini tekshirish:

python manage.py createsuperuser

‚ùáÔ∏è Modelni admin panelida ro'yxatdan o'tkazish:

üìå myapp/admin.py faylida:

from django.contrib import admin

admin.site.register(Task)

üìå http://127.0.0.1:8000/admin/ manzilida vazifalarni qo'shing yoki tahrirlang va loglarni tekshiring.

‚úÖ 13. SINOV UCHUN MASALALAR

üìå Celery vazifalarini sinash uchun quyidagi amallarni bajarib ko'ring: 1. /tasks/ manzilida yangi vazifa qo'shing va debug.log yoki Celery terminalida logni tekshiring. 2. Admin panelida vazifa qo'shing va Celery logini ko'ring. 3. Celery workerni o'chirib, vazifa qo'shing va logning kechiktirilganligini tekshiring. 4. /tasks/1/ manzilida vazifani yangilang va signal logini tekshiring.
