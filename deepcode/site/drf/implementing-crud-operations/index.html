üåê DJANGO REST FRAMEWORK ASOSLARI

üß© 3-DARS: CRUD OPERATSIYALARINI FUNCTION-BASED VIEWS BILAN AMALGA OSHIRISH

Bu darsda Django REST Framework (DRF) yordamida CRUD (Create, Read, Update, Delete) operatsiyalarini function-based views (FBV) yordamida bosqichma-bosqich amalga oshiramiz. Har bir qadam tushunarli va faqat CRUD operatsiyalariga qaratilgan bo'ladi. Oldingi darslarda sozlangan myproject loyihasi va myapp ilovasidagi Task modeli asosida davom etamiz.

‚úÖ 1. TAYYORLOV ISHLARI

üìå Loyiha va ilova allaqachon sozlangan deb hisoblaymiz (myproject, myapp, va Task modeli). Quyidagi sozlamalar mavjud bo'lishi kerak: - Task modeli myapp/models.py faylida belgilangan. - TaskSerializer myapp/serializers.py faylida yaratilgan. - Ma'lumotlar bazasi migratsiyalari qo'llanilgan. üìå Agar bu sozlamalar hali amalga oshirilmagan bo'lsa, avvalgi darslarga qayting.

‚úÖ 2. CRUD OPERATSIYALARINI TUSHUNISH

üìå CRUD operatsiyalari quyidagilarni o'z ichiga oladi: - Create: Yangi ma'lumot qo'shish. - Read: Mavjud ma'lumotlarni o'qish (ro'yxat yoki bitta ob'ekt). - Update: Mavjud ma'lumotlarni yangilash. - Delete: Ma'lumotlarni o'chirish.

üìå Function-based views yordamida har bir operatsiyani alohida funksiya sifatida yozamiz.

‚úÖ 3. MODEL VA SERIALIZERNI TEKSHIRISH

üìå Quyidagi sozlamalar oldingi darslarda amalga oshirilgan, lekin ularni qayta ko'rib chiqamiz:

‚ùáÔ∏è Model tekshirish:

üìå myapp/models.py faylida Task modeli quyidagicha bo'lishi kerak:

from django.db import models

class Task(models.Model):
    title = models.CharField(max_length=200)
    completed = models.BooleanField(default=False)
    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return self.title

‚ùáÔ∏è Serializer tekshirish:

üìå myapp/serializers.py faylida TaskSerializer quyidagicha bo'lishi kerak:

from rest_framework import serializers
from .models import Task

class TaskSerializer(serializers.ModelSerializer):
    class Meta:
        model = Task
        fields = ['id', 'title', 'completed', 'created_at']

‚úÖ 4. FUNCTION-BASED VIEWS YARATISH

üìå myapp/views.py faylini yangilaymiz va CRUD operatsiyalari uchun function-based views yozamiz:

from rest_framework.decorators import api_view
from rest_framework.response import Response
from rest_framework import status
from .models import Task
from .serializers import TaskSerializer

@api_view(['GET', 'POST'])
def task_list(request):
    if request.method == 'GET':
        tasks = Task.objects.all()
        serializer = TaskSerializer(tasks, many=True)
        return Response(serializer.data)

    elif request.method == 'POST':
        serializer = TaskSerializer(data=request.data)
        if serializer.is_valid():
            serializer.save()
            return Response(serializer.data, status=status.HTTP_201_CREATED)
        return Response(serializer.errors, status=status.HTTP_400_BAD_REQUEST)

@api_view(['GET', 'PUT', 'DELETE'])
def task_detail(request, pk):
    try:
        task = Task.objects.get(pk=pk)
    except Task.DoesNotExist:
        return Response(status=status.HTTP_404_NOT_FOUND)

    if request.method == 'GET':
        serializer = TaskSerializer(task)
        return Response(serializer.data)

    elif request.method == 'PUT':
        serializer = TaskSerializer(task, data=request.data)
        if serializer.is_valid():
            serializer.save()
            return Response(serializer.data)
        return Response(serializer.errors, status=status.HTTP_400_BAD_REQUEST)

    elif request.method == 'DELETE':
        task.delete()
        return Response(status=status.HTTP_204_NO_CONTENT)

üìå Tushuntirish: - task_list: GET (barcha vazifalarni o'qish) va POST (yangi vazifa qo'shish) so'rovlarini boshqaradi. - task_detail: GET (bitta vazifani o'qish), PUT (vazifani yangilash) va DELETE (vazifani o'chirish) so'rovlarini boshqaradi. - @api_view dekoratori DRFning API funksiyalarini yoqadi.

‚úÖ 5. URL SOZLASH

üìå myproject/urls.py faylini yangilaymiz va function-based views uchun URL marshrutlarini qo'shamiz:

from django.urls import path
from myapp.views import task_list, task_detail

urlpatterns = [
    path('tasks/', task_list, name='task-list'),
    path('tasks/<int:pk>/', task_detail, name='task-detail'),
]

üìå Tushuntirish: - /tasks/ manzili vazifalar ro'yxatini ko'rish va yangi vazifa qo'shish uchun. - /tasks/<pk>/ manzili bitta vazifani ko'rish, yangilash yoki o'chirish uchun.

‚úÖ 6. CRUD OPERATSIYALARINI SINAB KO'RISH

üìå Serverni ishga tushiring:

python manage.py runserver

üìå Brauzerda yoki Postman kabi vositalar yordamida quyidagi manzillarni sinab ko'ring:

‚ùáÔ∏è Create (Yangi vazifa qo'shish):

üìå Manzil: http://127.0.0.1:8000/tasks/
üìå Usul: POST
üìå Masalan (Postman orqali):

curl -X POST http://127.0.0.1:8000/tasks/ -H "Content-Type: application/json" -d '{"title": "Yangi vazifa", "completed": false}'

üìå DRF interfeysi orqali ham yangi vazifa qo'shishingiz mumkin.

‚ùáÔ∏è Read (Ma'lumotlarni o'qish):

  • Barcha vazifalarni ko'rish:
    üìå Manzil: http://127.0.0.1:8000/tasks/
    üìå Usul: GET
    üìå Brauzerda bu manzilga o'tsangiz, barcha vazifalar JSON formatida chiqadi.
  • Bitta vazifani ko'rish:
    üìå Manzil: http://127.0.0.1:8000/tasks/1/ (1 - vazifa ID si)
    üìå Usul: GET
    üìå Masalan: bash curl http://127.0.0.1:8000/tasks/1/

‚ùáÔ∏è Update (Ma'lumotlarni yangilash):

üìå Manzil: http://127.0.0.1:8000/tasks/1/
üìå Usul: PUT
üìå Masalan (vazifani yangilash uchun):

curl -X PUT http://127.0.0.1:8000/tasks/1/ -H "Content-Type: application/json" -d '{"title": "Yangilangan vazifa", "completed": true}'

‚ùáÔ∏è Delete (Ma'lumotlarni o'chirish):

üìå Manzil: http://127.0.0.1:8000/tasks/1/
üìå Usul: DELETE
üìå Masalan:

curl -X DELETE http://127.0.0.1:8000/tasks/1/

üìå Bu 1-ID li vazifani o'chiradi.

‚úÖ 7. ADMIN PANELI ORQALI SINOV (IXTIYORIY)

üìå Ma'lumotlarni qo'lda sinash uchun admin panelidan foydalaning:

‚ùáÔ∏è Admin foydalanuvchisini tekshirish:

üìå Agar hali yaratilmagan bo'lsa:

python manage.py createsuperuser

‚ùáÔ∏è Modelni admin panelida ro'yxatdan o'tkazish:

üìå myapp/admin.py faylida quyidagi kod bo'lishi kerak:

from django.contrib import admin
from .models import Task

admin.site.register(Task)

üìå http://127.0.0.1:8000/admin/ manzilida admin paneliga kiring va vazifalarni qo'shing, tahrirlang yoki o'chiring.

‚úÖ 8. CRUD SINOVI UCHUN MASALALAR

üìå API funksionalligini sinash uchun quyidagi amallarni bajarib ko'ring: 1. Brauzer orqali /tasks/ manzilida vazifalar ro'yxatini ko'ring. 2. Postman yoki DRF interfeysi yordamida yangi vazifa qo'shing (masalan, title: "Test vazifa", completed: False). 3. /tasks/1/ manzilida bitta vazifani yangilang (masalan, completed ni true ga o'zgartiring). 4. /tasks/1/ manzilidan foydalanib, 1-ID li vazifani o'chiring.
